== NGINX
Putty

Update the BELOW text with PORT and DOMAIN

server {
    listen 80;
    server_name bcweb.noodev8.com;

    location / {
        proxy_pass http://localhost:3020; # YOUR PORT
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Allow JSON content
        default_type application/json;
    }
}


-- Create the above file
sudo nano /etc/nginx/sites-available/bcweb.noodev8.com


-- Enable and reload
sudo ln -sf /etc/nginx/sites-available/bcweb.noodev8.com /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

-- Add SSL
sudo certbot --nginx -d bcweb.noodev8.com
SSL Done!

== Set Up production .env file
CORS is driven by CLIENT_URL, which already accepts a comma-separated list (server.js:
  (process.env.CLIENT_URL || 'http://localhost:3000').split(',')  ), so no code change is needed.

Copy windows contents from .env to clipboard
nano .env
Paste and update details
# CORS Configuration (comma-separated: local dev + the Vercel origin)
CLIENT_URL=http://localhost:3000,https://bcweb.vercel.app

# Environment
NODE_ENV=production

CTRL O, CTRL X
