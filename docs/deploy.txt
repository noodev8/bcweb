== Python (Amazon upload file feature) -- ASK CLAUDE FOR HELP WHEN DEPLOYING THIS
The "Amazon upload file" button (POST /product-amazon) shells out to a Python helper:
  bcweb-server/scripts/amz-product/amz_upload_single.py  (+ SHOES.xlsm template beside it)
The VPS is otherwise pure Node, so this needs a one-time setup on the server:
  - Python 3 installed and on PATH (or set PYTHON_BIN in the prod .env to the interpreter path)
  - pip packages: openpyxl, psycopg2 (or psycopg2-binary), python-dotenv
  - the scripts/amz-product/ folder must survive the rsync (it does today -- only 'docs' etc. are excluded)
  - the helper reads DB creds from bcweb-server/.env (already present in prod)
When you deploy this the first time, ask Claude to help wire up the Python env + verify the button end-to-end.


== Google Merchant price sync -- NOW IN-PROCESS NODE + a periodic SWEEP (ported off Python, 2026-07-24)
No Python, no venv, no google-price-push folder anymore. utils/googleMerchant.js pushes a salePrice-only supplemental override
to the Merchant API (products/v1) in-process; auth is utils/googleAuth.js (service-account JWT signed with Node's built-in crypto,
access token cached ~1h). Google is DECOUPLED from the apply: POST /pricing-apply just logs the change to price_change_log; a cron
SWEEP (scripts/google-price-sweep.js) sends the un-sent ones every ~2h through the day. Nightly merchant_feed.py --upload is the backstop.

Env (unchanged -- already in prod .env; NO new vars to add):
  GOOGLE_MERCHANT_ID, GOOGLE_MERCHANT_CREDENTIALS_JSON, GOOGLE_SUPPLEMENTAL_DATASOURCE=10688724635
  (optional GOOGLE_CONTENT_LANGUAGE=en / GOOGLE_FEED_LABEL=GB default to en/GB to match the feed -- leave unset)

DEPLOY STEPS (after the main rsync + npm install + pm2 restart above):
  1. Apply the DB migration in pgAdmin against brookfield_prod -- adds the sweep's queue marker AND backfills existing rows to now()
     so the first sweep only sends NEW changes (not the whole price history):
       bcweb-server/migrations/20260724_price_change_log_google_pushed_at.sql
  2. Schedule the sweep in cron -- every 2h through the working day, NOT at night.  crontab -e :
       0 8-20/2 * * *  cd /apps/production/bcweb-server && /usr/bin/node scripts/google-price-sweep.js >> /var/log/bcweb-google-sweep.log 2>&1
     - fix the node path to `which node` on the box; MUST cd into bcweb-server first (so .env + relative requires resolve).
  3. Smoke test: make a pricing change in the UI, then run once by hand:
       cd /apps/production/bcweb-server && node scripts/google-price-sweep.js
     Expect:  [google-sweep] pending=1 pushed=1 noop=0 failed=0 stamped=1   -- run it again -> "nothing pending" (proves dedup/stamp).

ONE-TIME ACCOUNT SETUP -- ALREADY DONE, DO NOT REPEAT and do NOT run per-machine:
  - GCP project registered with merchant account 124941872; "Merchant API" enabled in the Cloud project.
  - Supplemental data source id 10688724635 (account-level, permanent).
  - The supplemental source is LINKED into the PRIMARY feed's defaultRule.takeFromDataSources as
    [{supplementalDataSourceName: accounts/124941872/dataSources/10688724635}, {self:true}] (done 2026-07-24). WITHOUT this link
