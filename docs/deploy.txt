== Python (Amazon upload file feature) -- ASK CLAUDE FOR HELP WHEN DEPLOYING THIS
The "Amazon upload file" button (POST /product-amazon) shells out to a Python helper:
  bcweb-server/scripts/amz-product/amz_upload_single.py  (+ SHOES.xlsm template beside it)
The VPS is otherwise pure Node, so this needs a one-time setup on the server:
  - Python 3 installed and on PATH (or set PYTHON_BIN in the prod .env to the interpreter path)
  - pip packages: openpyxl, psycopg2 (or psycopg2-binary), python-dotenv
  - the scripts/amz-product/ folder must survive the rsync (it does today -- only 'docs' etc. are excluded)
  - the helper reads DB creds from bcweb-server/.env (already present in prod)
When you deploy this the first time, ask Claude to help wire up the Python env + verify the button end-to-end.


== Python (Google Merchant real-time price push) -- MIGRATED to the MERCHANT API on 2026-07-21
   (the old Content API for Shopping is discontinued by Google on 2026-08-18)
Applying a Shopify Pricing price (POST /pricing-apply, W1) shells out to a Python helper so Google Merchant Center's price
doesn't lag a full day behind the price change (the feed otherwise only refreshes via the nightly C:\scripts\merchant-feed\
merchant_feed.py --upload cron):
  bcweb-server/scripts/google-price-push/push_google_price.py
Uses the SAME PYTHON_BIN / venv as the Amazon upload feature above. TO DEPLOY THE MIGRATION ON THE VPS (do the main
rsync deploy first so the new files land, then):
  1. Install the new libs into the prod venv (replaces the old google-api-python-client Content API client):
       /apps/production/bcweb-server/venv/bin/pip install -r /apps/production/bcweb-server/scripts/google-price-push/requirements.txt
  2. Add ONE new line to the prod bcweb-server/.env (the supplemental data source, created once on 2026-07-21):
       GOOGLE_SUPPLEMENTAL_DATASOURCE=10688724635
     GOOGLE_MERCHANT_ID + GOOGLE_MERCHANT_CREDENTIALS_JSON are already present (same service-account creds) and carry over unchanged.
  3. pm2 restart bcweb-prod as usual, then verify by running the helper directly against a live-on-Google style
     (expect JSON like {"updated":14,"failed":0,"total":14}; it re-pushes the CURRENT price, so it changes no real price):
       /apps/production/bcweb-server/venv/bin/python /apps/production/bcweb-server/scripts/google-price-push/push_google_price.py 0051701-ARIZONA
  - the scripts/google-price-push/ folder must survive the rsync (only 'docs' etc. are excluded today)
